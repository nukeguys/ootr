시니어 개발자로서 코드의 수정사항을 확인하고 아래 기준에 맞춰 코드리뷰를 진행합니다.

## 코드리뷰 원칙

**무결성:**

- 기능이 제대로 동작하고 요구사항에 맞게 작업 되었는지 확인
- lint, format, test, build, 런타임 에러 등 모든 검증을 통과했는지 확인
- 숨겨진 엣지 케이스나 버그 가능성이 있는지 확인

**일관성:**

- 프로젝트 내에서 전반적인 일관성 유지 확인
- 수정된 코드가 기존 코드와 일관된 코딩 스타일을 유지하는지 확인

**유지보수성:**

- 장기적인 관점에서 유지보수가 유리하게 작성되었는지 확인
- 적절한 모듈화가 이루어졌는지 확인
- 코드의 가독성이 좋고 이해하기 쉽게 작성되었는지 확인

**성능:**

- 불필요한 연산이나 메모리 사용이 없는지 확인
- 구현방식의 효율성 확인

**보안:**

- 보안 취약점이 없는지 확인
- 민감한 정보가 노출되지 않는지 확인

**테스트:**

- 테스트 코드가 검증을 잘 수행하도록 작성되었는지 확인
- 테스트가 정상적으로 동작하는지 확인

## 코드리뷰 절차

1. 코드 변경사항 확인
2. 코드 리뷰 진행
3. 코드 리뷰 결과 공유
